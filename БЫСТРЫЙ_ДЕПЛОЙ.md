# 🚀 Быстрый деплой - Пошаговая инструкция

## Шаг 1: Загрузите код на GitHub (5 минут)

### Откройте терминал в папке проекта и выполните:

```bash
# 1. Инициализируйте git (если еще не сделано)
git init

# 2. Добавьте все файлы
git add .

# 3. Создайте первый коммит
git commit -m "Initial commit"

# 4. Подключите ваш GitHub репозиторий
git remote add origin https://github.com/dakia21/Dash.git

# 5. Переименуйте ветку в main
git branch -M main

# 6. Отправьте код на GitHub
git push -u origin main
```

**Если попросит логин/пароль:**
- Используйте Personal Access Token вместо пароля
- Создайте токен: GitHub → Settings → Developer settings → Personal access tokens → Generate new token

---

## Шаг 2: Настройте GitHub Pages (2 минуты)

1. Откройте ваш репозиторий: https://github.com/dakia21/Dash
2. Нажмите **Settings** (вверху справа)
3. В левом меню найдите **Pages**
4. В разделе **Source** выберите: **GitHub Actions**
5. Готово! GitHub автоматически начнет деплой

**Подождите 2-3 минуты** - GitHub Actions соберет и задеплоит ваш сайт.

**Ваш сайт будет доступен по адресу:**
```
https://dakia21.github.io/Dash/
```

---

## Шаг 3: Задеплойте сервер на Render.com (5 минут)

### 3.1. Зарегистрируйтесь на Render.com

1. Откройте: https://render.com
2. Нажмите **Get Started** или **Sign Up**
3. Выберите **Sign up with GitHub** (проще всего)
4. Разрешите доступ к вашим репозиториям

### 3.2. Создайте Web Service

1. На главной странице Render нажмите **New +**
2. Выберите **Web Service**
3. Найдите и выберите репозиторий **Dash**
4. Нажмите **Connect**

### 3.3. Настройте сервис

Заполните поля:

```
Name: dash-backend
Region: Frankfurt (EU Central)
Branch: main
Root Directory: server
Runtime: Node
Build Command: npm install
Start Command: node server-simple.js
Instance Type: Free
```

### 3.4. Добавьте переменные окружения

Прокрутите вниз до **Environment Variables** и нажмите **Add Environment Variable**

Добавьте 3 переменные:

```
NODE_ENV = production
JWT_SECRET = dash_super_secret_key_2024_change_this
PORT = 5001
```

### 3.5. Создайте сервис

1. Нажмите **Create Web Service** внизу страницы
2. Подождите 2-3 минуты пока Render задеплоит сервер
3. Скопируйте URL сервера (например: `https://dash-backend-xxxx.onrender.com`)

---

## Шаг 4: Обновите URL в проекте (3 минуты)

### 4.1. Обновите client/.env.production

Откройте файл `client/.env.production` и замените URL:

```env
VITE_API_URL=https://dash-backend-xxxx.onrender.com/api
VITE_SOCKET_URL=https://dash-backend-xxxx.onrender.com
```

**Замените `dash-backend-xxxx` на ваш реальный URL с Render!**

### 4.2. Обновите CORS в server/server-simple.js

Откройте файл `server/server-simple.js` и найдите строку ~10:

```javascript
const io = socketIo(server, {
  cors: {
    origin: [
      "http://localhost:5173",
      "https://dakia21.github.io"  // ← Добавьте эту строку
    ],
    methods: ["GET", "POST"]
  }
});
```

### 4.3. Сохраните и отправьте изменения

```bash
# Добавьте изменения
git add .

# Создайте коммит
git commit -m "Update production URLs"

# Отправьте на GitHub
git push
```

## 🐛 Если что-то не работает

### Сайт не открывается
1. Проверьте вкладку **Actions** на GitHub - должна быть зеленая галочка ✅
2. Подождите еще 2-3 минуты
3. Проверьте Settings → Pages - должен быть указан URL

### Не подключается к серверу
1. Откройте консоль браузера (F12)
2. Проверьте ошибки
3. Убедитесь, что сервер на Render работает (зеленый статус)
4. Проверьте URL в `.env.production`

### Сервер не запускается на Render
1. Откройте Logs в Render Dashboard
2. Проверьте ошибки
3. Убедитесь, что Root Directory = `server`
4. Убедитесь, что Start Command = `node server-simple.js`
