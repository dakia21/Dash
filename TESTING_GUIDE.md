# 🧪 Руководство по тестированию

## Как протестировать real-time сообщения

### Шаг 1: Запустите сервер
```bash
cd server
node server-simple.js
```

Вы должны увидеть:
```
🚀 Сервер запущен на порту 5001
✅ Работает в режиме in-memory (без MongoDB)
⚠️  Данные будут потеряны при перезапуске сервера
```

### Шаг 2: Запустите клиент
```bash
cd client
npm run dev
```

Откроется `http://localhost:5173`

### Шаг 3: Создайте двух пользователей

**Окно 1 (обычное окно браузера)**:
1. Зарегистрируйтесь как пользователь 1
   - Имя: `Алексей`
   - Email: `alex@test.com`
   - Пароль: `123456`

**Окно 2 (режим инкогнито)**:
1. Откройте `http://localhost:5173` в режиме инкогнито
2. Зарегистрируйтесь как пользователь 2
   - Имя: `Мария`
   - Email: `maria@test.com`
   - Пароль: `123456`

### Шаг 4: Начните чат

**В окне 2 (Мария)**:
1. В поле поиска введите `Алексей` или `alex`
2. Нажмите на найденного пользователя
3. Откроется чат

### Шаг 5: Проверьте real-time сообщения

**В окне 2 (Мария)**:
1. Напишите сообщение: `Привет, Алексей!`
2. Нажмите Enter или кнопку отправки

**В окне 1 (Алексей)**:
1. ✅ Сообщение должно появиться **СРАЗУ** без обновления страницы
2. Откройте консоль браузера (F12) и проверьте логи:
   ```
   ✅ Подключено к серверу
   📨 Получено сообщение через WebSocket: {...}
   ✅ Добавляем новое сообщение в чат
   ```

**В окне 1 (Алексей)**:
1. Ответьте: `Привет, Мария!`

**В окне 2 (Мария)**:
1. ✅ Ответ должен появиться **СРАЗУ**

## Проверка функций профиля

### Тест 1: Изменение цвета аватара

1. Нажмите на свой аватар в левом верхнем углу
2. Нажмите "Редактировать профиль"
3. Выберите любой цвет из палитры (10 цветов)
4. Нажмите "Сохранить"
5. ✅ Цвет аватара должен измениться везде:
   - В шапке sidebar
   - В списке чатов
   - В сообщениях чата

### Тест 2: Изменение displayName и username

1. Откройте профиль → "Редактировать профиль"
2. Измените "Отображаемое имя" на `Александр`
3. Измените "Имя пользователя" на `sasha`
4. Нажмите "Сохранить"
5. ✅ Проверьте:
   - В шапке показывается `Александр`
   - Под именем показывается `@sasha`
   - В чате собеседник видит `Александр`
   - Поиск работает по `sasha`

### Тест 3: Уникальность username

1. Попробуйте изменить username на уже занятый (например, на username другого пользователя)
2. ✅ Должна появиться ошибка: "Это имя пользователя уже занято"

## Проверка настроек

### Тест 1: Смена темы

1. Нажмите на иконку настроек (⚙️) в шапке sidebar
2. Переключите тему на "Темная"
3. ✅ Весь интерфейс должен стать темным
4. Переключите обратно на "Светлая"
5. ✅ Интерфейс должен вернуться к белому

### Тест 2: Смена цвета чата

1. Откройте настройки
2. Выберите другой цвет (например, фиолетовый)
3. ✅ Проверьте изменения:
   - Кнопка отправки сообщения
   - Ваши сообщения (синие пузыри)
   - Активный чат в списке
   - Переключатели (toggle)

### Тест 3: Размер шрифта

1. Откройте настройки
2. Увеличьте размер шрифта (кнопка с большой буквой T)
3. ✅ Текст во всем приложении должен стать крупнее
4. Уменьшите обратно

## Известные проблемы и решения

### Проблема: Сообщения не приходят в реальном времени

**Решение**:
1. Проверьте консоль браузера - должно быть "✅ Подключено к серверу"
2. Если нет подключения, перезапустите сервер
3. Обновите страницу в браузере (F5)
4. Проверьте, что сервер запущен на порту 5001

### Проблема: Пользователь не находится в поиске

**Решение**:
1. Убедитесь, что пользователь зарегистрирован
2. Проверьте логи сервера - должно быть "✅ ПОЛЬЗОВАТЕЛЬ СОЗДАН"
3. Попробуйте искать по email вместо имени
4. Перезапустите сервер (данные в памяти могут быть потеряны)

### Проблема: Цвет аватара не меняется

**Решение**:
1. Проверьте, что вы нажали "Сохранить" в профиле
2. Обновите страницу (F5)
3. Проверьте консоль на ошибки
4. Убедитесь, что сервер вернул успешный ответ

### Проблема: Дублирование сообщений

**Решение**:
- Эта проблема должна быть исправлена
- Если все еще дублируются, проверьте консоль браузера
- Сообщения от себя не должны добавляться через WebSocket
