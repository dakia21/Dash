## 🎯 ЧТО ДЕЛАТЬ СЕЙЧАС:

1. Откройте http://localhost:5173

3. Но регистрация не будет работать без MongoDB

### Вариант 2: Полный запуск (РЕКОМЕНДУЕТСЯ)

#### Шаг 1: Настройте MongoDB Atlas (5 минут)

**Самый простой способ - облачная БД:**

1. Зайдите на https://www.mongodb.com/cloud/atlas/register
2. Создайте бесплатный аккаунт
3. Создайте кластер (FREE M0)
4. Database Access → Создайте пользователя (username + password)
5. Network Access → Add IP → 0.0.0.0/0 (разрешить все)
6. Connect → Connect your application → Скопируйте строку подключения

**Пример строки:**
```
mongodb+srv://myuser:mypassword@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority
```

#### Шаг 2: Обновите конфигурацию

Откройте файл `server/.env` и замените строку MONGODB_URI:

```env
MONGODB_URI=mongodb+srv://ваш_username:ваш_password@cluster0.xxxxx.mongodb.net/messenger?retryWrites=true&w=majority
```

⚠️ **Важно**: Замените `ваш_username` и `ваш_password` на реальные данные!

#### Шаг 3: Перезапустите сервер

Сервер автоматически перезапустится (nodemon) или перезапустите вручную.

#### Шаг 4: Проверьте подключение

В логах сервера должно появиться:
```
✅ MongoDB подключена
🚀 Сервер запущен на порту 5000
```

## ❓ Частые вопросы:

**Q: Почему не работает регистрация?**  
A: Нужно настроить MongoDB (см. Шаг 1 выше)

**Q: Как добавить второго пользователя для теста?**  
A: Откройте приложение в режиме инкогнито и зарегистрируйте еще один аккаунт

**Q: Сообщения не приходят мгновенно?**  
A: Проверьте что оба пользователя онлайн и WebSocket подключен

**Q: Можно ли использовать без MongoDB?**  
A: Нет, MongoDB необходима для хранения пользователей и сообщений
